# RAG 查询分层路由策略

## 文档目标
本文档说明不同类型用户问题应该选择不同 RAG 检索链路，避免所有查询默认走复杂链路造成排序退化和延迟上升。

## 简单事实类
定义、口径、单一政策解释属于简单事实类问题。推荐使用 Dense Retrieval 快路径，必要时补充相邻 chunk 作为上下文，不默认启用 MQE 和 LLM Rerank。

## 关键词政策类
包含明确制度名、等级、编号、SOP、P1、FCR、SLA、SKU 等关键词的问题，适合启用 Dense + BM25 + RRF。BM25 可以增强精确术语召回，但需要通过评估确认是否引入关键词噪声。

## 多文档综合类
同时涉及会员、促销、物流、风控、退款、投诉等多个条件的问题，适合启用 MQE + Hybrid Retrieval。MQE 的作用是把隐含条件展开成多个检索角度。

## 复杂诊断类
需要解释指标异常、归因分析、风险判断或规则冲突的问题，可以在 Hybrid Retrieval 后追加 LLM Rerank，但必须记录延迟和排序质量变化。若 Rerank 导致 MRR 或 NDCG 下降，应回退到轻量链路。

## 不清晰问题
缺少时间、类目、指标或业务对象的问题，应先进入 clarification 节点追问，不应强行检索。
